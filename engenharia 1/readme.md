A engenharia de software é intangível. 
- A engenharia de software se difere das outras, tangíveis, uma vez que ela cria o que não se pode ver e tocar, o software está presente em muitos cenários atualmente,
 mas nem por isso é notado com tanta facilidade quanto se nota um edifício. Além disso, suua teoria é menos rígida e mais fluída.

A engenharia de software é igual à programação + tempo + escabilidade e trade-off´s.
- Base teórica planejada, desenvolvida, testada e consolidada criam os princípios da engenharia de software que, inegávelmente, surgem da programação e do bom uso
do tempo. Essa engenharia possui ferramentas e recursos para construir sistemas e projetos organizados através da prática.

Trade-Off
- Os trade-off´s são requisitos não funcionais que garantem a funcionabilidade dos códigos ao longo dos anos com uma condição de ganha-perde, gerando equilíbrio
entre duas qualidades conflitantes.
